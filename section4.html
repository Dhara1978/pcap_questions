Section 4: Object-Oriented Programming (34%)

Understand the Object-Oriented approach

  • Class: A blueprint for creating objects
  • Object: An instance of a class
  • Property: Characteristics of an object
  • Method: Functions defined inside a class
  • Encapsulation: Bundling data and methods within a class
  • Inheritance: Mechanism where one class inherits properties and methods from another
  • Superclass: The class being inherited from
  • Subclass: The class that inherits from the superclass
  • Identifying class components: Recognizing attributes and methods in a class

    Employ class and object properties

    • Instance vs. Class Variables:
      • Instance variables are unique to each object
      • Class variables are shared among all instances of a class
    • The __dict__ Property:
      • For objects, it holds all instance attributes
      • For classes, it holds class-level attributes
    • Private Components:
      • Instances: Prefixed with __, they are inaccessible from outside the class
      • Classes: Similar to instance attributes but on a class level
    • Name Mangling: Internally changes private attribute names to avoid conflicts

Equip a class with methods

  • Declaring and Using Methods: Define functions within a class that operate on its instances
  • The self Parameter: Refers to the instance calling the method

Discover the class structure

  • Introspection: Examining the properties and capabilities of an object at runtime
  • The hasattr() Function: Checks if an object has a particular attribute
  • Properties:
    • __name__: The name of the class
    • __module__: The module in which the class is defined
    • __bases__: A tuple of base classes

Build a class hierarchy using inheritance

  • Single and Multiple Inheritance: Inheriting from one or multiple classes
  • The isinstance() Function: Checks if an object is an instance of a class or a tuple of classes
  • Overriding: Redefining a method in a subclass
  • Operators:
    • not is: Checks if two variables do not reference the same object.
    • is: Checks if two variables reference the same object
  • Polymorphism: Methods can perform different tasks based on the object calling them
  • Overriding the __str__() Method: Customizes the string representation of an object
  • Diamonds: Refers to the Diamond Problem in multiple inheritance
    • Imagine you have four classes in the following hierarchy:
    • Class A: The top-level base class
    • Class B and Class C: Both inherit from A
    • Class D: Inherits from both B and C
    • Problem Description
    • The issue arises when Class D tries to access methods or attributes that are defined in
    • Class A. Because D inherits from both B and C, and both B and C inherit from
    • A, there can be ambiguity in which version of the method or attribute to use from A

Construct and initialize objects

  • Declaring and Invoking Constructors: Special methods (__init__) that initialize object attributes
